# 2026-05-29 Memory

## Vercel Deployment Crisis (2026-05-28 04:00 GMT+8)

### The Problem
- Builds failing with "Function size exceeds 250MB limit" for ~20 hours
- Root cause: `.vercelignore` misconfigurations

### `.vercelignore` Lessons
- `*.png` wildcard blocks ALL PNGs everywhere → changed to `/*.png` (root only)
- `memes/` without leading slash matches ANY `memes/` directory → changed to `/memes/` (root only)
- This was excluding `public/memes/` from deployment, breaking the art gallery

### Fix
- `.vercelignore` updated with proper root-level patterns:
  - `/*.png` instead of `*.png`
  - `/memes/` instead of `memes/`
  - `/__pycache__/` instead of `__pycache__/`
- Deploy package dropped from 256MB to ~36MB

### Duplicate Artwork Issue
- mad-art gallery had duplicate entries in `artworks` array:
  - `mad-cyber-troll.jpg` = `mad-cyber-troll-v2.jpg`
  - `mad-green-pill.jpg` = `mad-jungle-pill.jpg`
  - `mad-green-pill-v3.jpg` = `mad-jungle-pill.jpg`
  - `mad-morning-routine.jpg` = `mad-morning-cereal.jpg`
- Fixed by removing 4 duplicates, gallery now shows 34 unique pieces
