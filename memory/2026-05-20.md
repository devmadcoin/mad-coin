**Core technique**: Blueprint-based modeling

**Key principles for better car modeling**:
1. **Use blueprints/reference** — Don't model from imagination; use side/top/front reference images
2. **Low-poly workflow** — Optimized for real-time/game use (perfect for web)
3. **Hard-surface topology** — Cars need clean edge flow, not sculpting
4. **Proper proportions** — Wheelbase, overhang, cabin height ratios matter
5. **Symmetry** — Use mirror modifier, model one side
6. **Stylized > realistic** — Low-poly aesthetic is intentional and clean
7. **Blueprint setup** — Import reference images as background planes in orthographic views

**What the current MAD car is missing**:
- No reference/blueprints used — proportions are guessed
- Body is just a stretched cube with subdivision — no actual car shape
- No wheel wells/arches — wheels just sit next to the body
- Headlights are spheres glued on — not integrated into body shape
- No proper hood curve, windshield rake, or cabin profile
- Spoiler is a rectangle — no aerodynamic shape

## Wheel Spin Fix (12:23 PM)

**User feedback**: "now the car is in the correct direction but the way the wheels spin is off"

**Root cause**: The tire torus was lying flat on the ground (default Blender orientation), so rotating it made it tumble instead of spin like a real wheel.

**Fix**:
- In Blender script: rotated tires 90° on Y axis → stand upright like real wheels
- Aligned rims to same axle axis → both spin together
- Shrunk wheel size from 0.22/0.17 to 0.11/0.07 (was oversized)
- In React `MadChao3D.tsx`: `wheelRefs` now includes both `Wheel_*` and `Rim_*` objects, spinning at same rate
- Commit: `1776931` — wheel orientation fix, spin both tire and rim

**Lesson**: When building wheels in Blender for Three.js, always check the default orientation. A torus primitive lies flat by default — must rotate 90° on Y to make it roll like a wheel.
