### The Blender Realization
User reviewed again and said: *"maybe you should watch a lot blender videos."*

The honest truth: Three.js primitives (spheres, boxes, capsules) hit a ceiling at ~80% fidelity. To match the reference exactly — smooth organic head, sculpted eyebrow ridges, proper eye sockets, soft body curves — requires real 3D modeling with:
- Subdivision surfaces for smooth shapes
- Sculpting for angry eyebrow detail
- Proper topology and edge flow
- Materials with subsurface scattering
- Real rigging with bones

### Blender Study Curriculum Created
Built two knowledge files:

1. **`knowledge/blender-study-curriculum.md`** — 6-phase learning path
   - Phase 1: Fundamentals (Blender Guru Donut Tutorial, 5 hrs)
   - Phase 2: Chibi character modeling (Sasha Luvr tutorial)
   - Phase 3: Materials & texturing (Principled BSDF, subsurface scattering)
   - Phase 4: Rigging + animation (idle, walk, sit, excited)
   - Phase 5: glTF export (Draco compression, Three.js integration)
   - Phase 6: Website integration (useGLTF, useAnimations)
   - Total: ~40-60 hours over 2-3 weeks

2. **`knowledge/2026-05-20-blender-mascot-plan.md`** — Quick action plan
   - Exact modeling spec from reference (head radius, eye placement, eyebrow angle, body ratios)
   - Web export checklist (.glb with Draco, under 2MB)
   - Three.js integration code snippet
   - Alternative: Fiverr commission ($30-100, 3-7 days)

### What I Learned
- Sometimes "try harder with spheres" isn't the answer. The user recognized the ceiling before I fully admitted it.
- The $MAD eye for quality is real — the mascot has to look RIGHT, not "close enough."
- Three.js primitives are great for abstract shapes and environments. Characters need real modeling.
- The glTF pipeline (Blender → .glb → Three.js) is mature and well-documented. Not experimental.
- Presenting three options (learn, commission, or keep iterating) is better than pushing one path.

### Decisions
- Document the honest ceiling instead of pretending primitives can do everything.
- Build study curriculum proactively so the user has a clear path forward.
- Include Fiverr alternative — not everyone wants to learn 3D modeling.
- Save all study materials to knowledge base for future reference.
