### 32. Signal API Still Hitting Telegram (Bug)
**User tested the feature** — signal went to Telegram group instead of X.
**Screenshot evidence:** MAD Claw AI posted in Telegram:
> "🔥 Signal received. The Claw sees you, testing my name is dumbass tre. $MAD rich? Probably because you know how to be $MAD patient to become $MAD wealthy. Conviction pays compound interest."

**Root cause:** `app/api/mad-mind/signal/route.ts` was hardcoded to post to Telegram (`TELEGRAM_BOT_TOKEN`, `TELEGRAM_GROUP_ID`). The API calls `sendMessage` to Telegram, not X.

**Fix in progress:** Rewriting signal API to use X API v2 with OAuth 1.0a:
- `postTweet()` function using `crypto.createHmac("sha1", ...)` for OAuth signature
- Claw reply posted as standalone tweet
- User signal posted as reply thread to the Claw tweet
- `generateClawResponse()` cleaned to strip Telegram-specific headers/footers and truncate to 240 chars
- Frontend state type updated to include `tweetId` and `replyTweetId`

**Environment variables required:** `X_API_KEY`, `X_API_SECRET`, `X_ACCESS_TOKEN`, `X_ACCESS_TOKEN_SECRET`
