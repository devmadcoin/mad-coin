# 2026-05-16 — X Bot Repetitive Content Fix

**Time:** 01:23 AM (Asia/Shanghai)

## Problem
User reported the X bot was still posting the same repetitive content — specifically:
- "The $MAD framework in 3 parts" thread (posted at both 5h and 2h ago)
- "Someone asked: 'Why $MAD?'" thread (posted 8h ago)
- Same exact phrases like "You don't need more alpha. You need less emotion."

Screenshots confirmed identical threads being posted hours apart.

## Root Cause
Found the bug in `/root/.openclaw/workspace/mad_x_bot.py`:

1. **`generate_thread_candidates()`** used hardcoded static thread arrays (~20 pre-written threads) instead of the dynamic synthesis engine. Lines ~1305-1332 contained the same "3 parts" and "Why $MAD" threads that kept cycling.

2. **Deduplication was failing** because:
   - `thread_fingerprint()` and `normalize_for_dedup()` existed but only blocked exact matches
   - The 7-day cooldown in `blocked_fingerprints` was being checked, but the static pool was so small that threads were still repeating within hours
   - The `recent_norm_fingerprints` check only looked at the last 60 texts, but with a 2h posting cycle, that wasn't enough to prevent "2h ago" vs "5h ago" duplicates

3. **Philosophy synthesis was only 60%** — fallback to `_generate_legacy_philosophy_candidates()` was frequently hit, which used `CLAUSE_A` + `CLAUSE_B` pools with repetitive phrases.

4. **`MAD_SPECIFIC_TEMPLATES` and `CELEBRATION_TEMPLATES`** were static pools with ~20 entries each, causing repetition in single posts too.

## Fix Applied
1. **Replaced static thread generator** — `generate_thread_candidates()` now builds every thread dynamically from the knowledge base, combining 1-3 frameworks into fresh 2-4 tweet threads. Removed all hardcoded thread arrays.

2. **Replaced static single-post pools** — `generate_mad_specific_candidates()` and `generate_celebration_candidates()` now use the synthesis engine instead of `MAD_SPECIFIC_TEMPLATES` and `CELEBRATION_TEMPLATES`.

3. **Increased philosophy synthesis to 90%** — `generate_philosophy_candidates()` now tries dynamic synthesis for 90% of posts. Legacy fallback only triggers if synthesis fails completely.

4. **Purged repetitive phrases** from legacy fallback pools — removed exact duplicates like "You don't need more alpha. You need less emotion." from `CLAUSE_A` and `CLAUSE_B`.

5. **Restarted the bot** — new code running (PID 198038).

6. **Committed and pushed** — `main` branch updated on GitHub.
