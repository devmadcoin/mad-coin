## 1. JOSH GAMBRELL — Hard Surface Boolean Workflow

**Core Philosophy:** "Don't model details — cut them in"

**The Boolean Stack Method:**
1. **Start with primitives** — cubes, cylinders for base shape
2. **Use Boolean Difference** to cut holes, panels, vents
3. **Use Boolean Union** to add protrusions, bolts, greebles
4. **NEVER apply booleans immediately** — keep them live in modifier stack
5. **Bevel AFTER booleans** — Ctrl+B on sharp edges (0.002-0.005 segments)
6. **Separate bevel objects** — create a "chamfer cube" rotated 45°, boolean it against edges for perfect 45° cuts
7. **Apply order:** Mirror → Array → Booleans → Bevel → Subdivision

**Key Settings:**
- Boolean Solver: Exact (for clean cuts) or Fast (for speed)
- Bevel: Width 0.003-0.01, Segments 2-3, Profile 0.5
- Use "Harden Normals" on bevel modifier

**For MAD Car Application:**
- Panel lines: boolean thin cubes as cutting objects
- Headlight housings: boolean cylinders into front fenders
- Wheel arches: boolean half-torus shapes
- Exhaust ports: boolean small cylinders
- Door seams: boolean intersecting planes

## 2. GLEB ALEXANDROV — Bevel Modifier Stack

**Core Philosophy:** "Perfect edges make any model look expensive"

**The Weighted Normals Stack:**
1. **Bevel Modifier #1** — Width 0.003, Segments 3, Profile 0.5
2. **Weighted Normal Modifier** — Weighting Mode: Face Area + Angle
3. **Bevel Modifier #2** — Width 0.0005, Segments 2 (for micro-bevels)
4. **Subdivision Surface** — Level 2 (only at final render)

**Why This Works:**
- Weighted normals prevent "pinched" shading at bevels
- Face area weighting = larger faces get more normal influence
- Angle weighting = sharper corners get stronger normals
- Stack order matters: Bevel → Weighted Normal → Subsurf

**Key Insight:**
- Without weighted normals, beveled edges get dark artifacts
- Weighted normal modifier fixes this automatically
- Use "Keep Sharp" option to preserve intentional sharp edges

**For MAD Car Application:**
- All body edges get bevel + weighted normals
- Wheel arches: bevel width 0.005 for soft rounded edges
- Hood crease lines: mark as SHARP (Ctrl+E → Mark Sharp), bevel skips them
- Headlight rim: bevel width 0.002 for tight edge

## 3. FLYCAT — Sculpting & Retopology Workflow

**Core Philosophy:** "Form first, topology second"

**The Full Pipeline:**
1. **Base mesh** — simple cube or sphere, 6-8 divisions
2. **Sculpt mode** — grab, draw, inflate to get basic form
3. **Dynamic topology** — enable for freedom, disable for control
4. **Major forms first** — don't detail until proportions are right
5. **Retopology** — build clean mesh OVER the sculpt:
   - Add new mesh (plane)
   - Shrinkwrap modifier to sculpt
   - Build edge loops around eyes, mouth, joints
   - Mirror modifier for symmetry
   - Apply shrinkwrap when clean
6. **Shape keys** — test expressions before finalizing
7. **UV unwrap** — after locked topology

**Key Brushes (in order of use):**
1. Grab (G) — move large masses
2. Draw (D) — add mass
3. Inflate (I) — puff up areas
4. Smooth (S) — blend transitions
5. Crease (C) — sharp edges
6. Pinch (P) — tight corners

**Retopology Rules:**
- Edge loops around eyes: 2 rings minimum (eyelid fold)
- Mouth: 3 loops (upper lip, middle, lower lip)
- Joints: extra loops for bending (elbow = 3 loops)
- Avoid poles > 5 edges near deformation areas
- Quads ONLY on deformation surfaces

**For MAD Mascot Application:**
- Sculpt organic head shape first (sphere → push cheeks → flatten top)
- Retopologize with proper edge flow for expressions
- Shape keys: smile, angry, neutral
- The body can stay low-poly (no deformation needed)

## 4. IAN HUBERT — Lazy Cinematic Techniques

**Core Philosophy:** "If you can't see it, don't model it"

**The Lazy Toolkit:**
1. **Camera-mapped projection** — project photos onto simple geometry
2. **Kitbashing** — reuse the same 3 objects in different arrangements
3. **Simple geometry** — cubes with displacement = city buildings
4. **Photo planes** — 2D images with alpha in 3D space
5. **Distance = blur** — atmospheric perspective hides low detail
6. **Particle systems** — 1 object instanced 1000x = crowd/city
7. **Texture bombing** — reuse textures at different scales/rotations

**Key Techniques:**
- **Projection mapping:** Camera track footage → project onto reconstructed geometry → fill with new elements
- **De-lighting:** Remove original lighting from projected textures → relight in 3D
- **The "Salad Mug" approach:** Simple geometry + good textures + atmospheric fog = cinematic shot
- **Grease pencil for annotations:** Draw directly on 3D viewport

**For MAD Project Application:**
- Environment: simple neon planes + fog (no complex modeling)
- City background: instanced cubes with emission shaders
- Track surface: single plane with neon stripe texture
- Particle exhaust: 1 sphere instanced with emission
- Camera: always at dramatic low angle (cinematic)
- Fog/mist: hides distant geometry, adds depth

## 5. DUCKY 3D — Modifier Stack Art

**Core Philosophy:** "Let modifiers do the work"

**The Stack Architecture:**
1. **Array Modifier** — Repeat object along axis (spokes, lights, bolts)
2. **Curve Modifier** — Bend array along path (neon tubes, piping)
3. **Mirror Modifier** — Always on TOP of stack (below = broken symmetry)
4. **Boolean Modifier** — Add/subtract detail objects
5. **Solidify Modifier** — Give thickness to flat geometry
6. **Wireframe Modifier** — Convert faces to wire (hologram effect)
7. **Subdivision Surface** — Last in stack (smooths everything)

**Signature Combinations:**
- Array + Curve = repeating elements along path (exhaust piping, light strips)
- Array + Mirror = radial symmetry (wheel spokes, turbine blades)
- Boolean + Bevel = cutouts with soft edges (vents, grilles)
- Solidify + Wireframe = tech/ hologram look

**For MAD Car Application:**
- Wheel spokes: Array (8x) + Mirror (radial)
- Neon underglow: Array + Curve along car bottom
- Grille mesh: Array (horizontal) + Array (vertical) + Boolean
- Side vents: Boolean cut + Bevel modifier
- Dashboard gauges: Array in grid pattern

## 🎯 Blender API Mastery Log

**What I learned about bpy today:**

1. **Proportional editing programmatically:**
   ```python
   bpy.context.scene.tool_settings.use_proportional_edit = True
   bpy.context.scene.tool_settings.proportional_edit_falloff = 'SMOOTH'
   bpy.context.scene.tool_settings.proportional_size = 0.3
   bpy.ops.transform.translate(value=(0, 0.1, 0), proportional_size=0.25)
   ```
   - Must enable before transform, disable after
   - Size controls brush radius in Blender units

2. **Boolean modifier live stack:**
   ```python
   mod = obj.modifiers.new(name="Boolean", type='BOOLEAN')
   mod.operation = 'DIFFERENCE'
   mod.object = cutter
   mod.solver = 'FAST'  # or 'EXACT'
   ```
   - Cutter object must exist in scene (hidden is fine)
   - Export applies modifiers automatically
   - Multiple booleans stack in order added

3. **Weighted normal modifier:**
   ```python
   wn = obj.modifiers.new(name="WeightedNormal", type='WEIGHTED_NORMAL')
   wn.weight = 50
   wn.mode = 'FACE_AREA'
   wn.keep_sharp = True
   ```
   - Always AFTER bevel in stack
   - `keep_sharp = True` preserves intentional creases
   - Fixes 90% of shading artifacts on beveled hard-surface

4. **Array + Empty for radial patterns:**
   ```python
   array = obj.modifiers.new(name="RadialArray", type='ARRAY')
   array.count = 5
   array.use_relative_offset = False
   array.use_object_offset = True
   array.offset_object = empty
   empty.rotation_euler = (0, 0, 2 * math.pi / count)
   ```
   - Empty acts as pivot point
   - Each copy inherits rotation/scale from empty
   - Classic Ducky 3D wheel spoke technique

5. **Mark sharp edges via API:**
   ```python
   bpy.ops.object.mode_set(mode='EDIT')
   bpy.ops.mesh.select_mode(type='EDGE')
   bpy.ops.mesh.edges_select_sharp(sharpness=math.radians(60))
   bpy.ops.mesh.mark_sharp(clear=False)
   bpy.ops.object.mode_set(mode='OBJECT')
   ```
   - Bevel modifier with `limit_method='ANGLE'` respects these
   - Creates intentional hard creases on otherwise smooth surfaces

6. **glTF export with collection filtering:**
   ```python
   bpy.ops.object.select_all(action='DESELECT')
   for obj in coll.objects:
       obj.select_set(True)
   bpy.ops.export_scene.gltf(use_selection=True, ...)
   ```
   - Only exports objects in target collection
   - Hidden boolean cutters excluded
   - `export_draco_mesh_compression_enable=True` keeps files small

## 1. UV UNWRAPPING & TEXTURE MAPPING

### What UV Unwrapping Is
- UV = U (horizontal) + V (vertical) coordinate system
- Projects 3D surface onto 2D plane so image textures can be painted/mapped onto the model
- Like taking a cardboard box apart to paint it flat, then folding it back up

### Manual UV Unwrapping Workflow
1. **Select edges to mark as SEAMS** — where the "cut" will happen when unfolding
2. **Mark Seam** → Ctrl+E → Mark Seam (or in UV menu)
3. **Select entire mesh** → U key → choose unwrap method:
   - **Unwrap**: Standard, follows seams
   - **Smart UV Project**: Automatic for complex meshes
   - **Follow Active Quads**: For long strips (hair, belts)
   - **Sphere/Cylinder/Cube Projection**: Primitive shapes

### UV Strategy Rules
- **Use as few seams as possible** — more seams = harder to texture
- **Hide seams where they won't be seen** — inside legs, armpits, under arms
- **Minimize stretching** — check with the UV Stretching overlay (blue = good, red = bad)
- **Keep islands proportional** — don't distort the scale too much between islands
- **Leave margin between islands** — padding for texture bleeding, especially for baked textures
- **Align to axes** — for mechanical objects, straight UV edges make texturing easier

### UV Layout Best Practices
- Pack efficiently — minimize wasted texture space
- Place big/island details in center — edge areas get mip-mapped worse
- Mirror UVs for symmetrical objects to save texture space
- Use UV pinning (P key) to lock island positions while packing others
- Blender's "Pack Islands" (Ctrl+P) auto-packs with configurable margin

### Texture Painting in Blender
- **Texture Paint workspace** → Setup image texture in material
- Paint directly on model with brushes (cloning, smearing, color mixing)
- Use stencils and masks for controlled painting
- Export image (UV/Image Editor → Image → Save As) for external editing in Photoshop/GIMP/Krita

## 2. PROCEDURAL TEXTURING & CAR PAINT SHADERS

### Procedural vs Image Textures
- **Procedural**: Generated by math (noise, voronoi, waves). Infinite resolution, no UVs needed.
- **Image**: Bitmap file mapped via UV. Needs UVs, limited resolution.
- Best workflow: Use procedural for base variation, image textures for logos/decals

### Key Procedural Texture Nodes

**Noise Texture**: Organic variation, clouds, dirt, subtle color shifts
- Scale: controls frequency
- Detail: how many octaves of detail
- Roughness: how chaotic
- ColorRamp after it to map values to colors

**Voronoi Texture**: Cell patterns, scales, cracked earth, sci-fi panels
- F1-F2 distance mode for cracked effect
- Coloring for cell variation

**Wave Texture**: Stripes, wood grain, ripples
- Rings mode for wood
- Bands mode for zebra/zebra-like

**Musgrave Texture** (Blender 4.1+ merged into Noise Texture):
- fBM: fractal brownian motion (clouds, fog)
- Multi-Fractal: more dramatic variation
- Note: In Blender 4.1+, Musgrave is gone. Use Noise Texture with Musgrave type, or Voronoi

### Building a Car Paint Material (Procedural)
1. **Base coat** → Principled BSDF:
   - Base Color: Deep red (#D00000)
   - Metallic: 0.8 (clearcoat metallic)
   - Roughness: 0.1-0.2
   - Clearcoat: 1.0 (the glossy top layer)
   - Clearcoat Roughness: 0.03 (super glossy)

2. **Color variation layer** → Noise Texture (Scale 50, Detail 15):
   - ColorRamp: Input 0 = red, Input 1 = darker red
   - Mix with base color using MixRGB (Factor 0.2)
   - Creates subtle metallic flake variation

3. **Flake/sparkle layer** → Voronoi Texture (Scale 200, Randomness 0.5):
   - Multiply by 0.1
   - Add to base color for metallic flake effect

4. **Dirt/edge wear** → Geometry node (Pointiness) + ColorRamp:
   - Pointiness detects concave/convex edges
   - ColorRamp: concave areas = dark brown dirt
   - Mix with factor 0.15 for subtle edge darkening

5. **Emissive accents** → Separate emission shader:
   - Neon strips: Emission strength 5-10, cyan (#00FFFF)
   - Only on specific faces or via vertex color masking

### Normal Maps from Procedural Textures
- Bump node converts grayscale height to normal perturbation
- Noise → Bump → Normal input of Principled BSDF
- Strength controls how deep the bump appears
- For fine surface detail (brushed metal, orange peel clearcoat)

## 3. SCULPTING — DYNTOPO vs MULTIRESOLUTION

### Sculpt Mode Basics
- **Sculpt Mode**: Edit mesh by pushing/pulling like digital clay
- Brush shortcut: F key (resize), Shift+F (strength)
- Common brushes: Draw, Smooth, Grab, Inflate, Crease, Pinch

### Dynamic Topology (Dyntopo)
- **What it does**: Automatically subdivides where you sculpt, keeps low poly elsewhere
- **Best for**: Concept sculpting, organic shapes, when topology doesn't matter
- **Pros**: No performance ceiling, add detail anywhere
- **Cons**: Destroys topology, messy mesh, hard to UV unwrap, can't use smooth shading well
- **Use case**: First pass rough sculpt, then retopology later

### Multiresolution Modifier (Multires)
- **What it does**: Creates subdivision levels you can sculpt on
- **Best for**: Final detail sculpting after good topology exists
- **Pros**: Clean topology preserved, bake normal/displacement maps, animate after sculpting
- **Cons**: Requires good base topology first, subdivides ENTIRE mesh (performance cost)
- **Workflow**: Sculpt low level = big shapes → higher level = medium detail → highest = skin pores

### The Complete Sculpting Workflow
1. **Blockout** (low poly, no sculpt) → get proportions right
2. **Dyntopo rough sculpt** → form and silhouette
3. **Retopology** → clean quads for animation (use Retopoflow addon or manual)
4. **Add Multires modifier** → subdivide 3-5 levels
5. **Sculpt final details** on multires level 4-5
6. **Bake** normal map from multires to base mesh
7. **Use base mesh + normal map** for animation/game engines

### Key Sculpting Tips
- Sculpt with symmetry on (X mirror) for characters
- Use Grab brush for large shape changes (move whole nose, reposition ear)
- Use Clay brush for additive volume
- Use Smooth (Shift key while sculpting) to blend transitions
- Use Crease for sharp edges (eyelids, lips)
- Alpha textures for skin pores, scales, fabric weave
- Masking (Ctrl+click drag to mask, Alt+mask to clear) to protect areas

## 4. NORMAL MAP BAKING — HIGH POLY TO LOW POLY

### Why Bake Normal Maps
- Game engines can't render million-poly sculpts in real-time
- Normal map stores surface direction ("fake" geometry detail)
- Low poly (5K tris) + normal map = looks like high poly (1M tris)

### Baking Workflow
1. **Create high poly** (sculpted with multires or dyntopo)
2. **Create low poly** (retopologized or decimated base mesh)
3. **UV unwrap the LOW poly** — this is where the texture bakes
4. **Create image texture** (2048x2048 or 4096x4096)
5. **Add image texture to low poly material** (must be selected/active)
6. **Align both models** — high poly should surround low poly
7. **Select high poly → Shift+select low poly** (low poly = active object)
8. **Render Properties → Bake**:
   - Bake Type: Normal
   - Selected to Active: ON
   - Ray Distance: 0.1-0.5 (how far rays shoot from low to high)
   - Extrusion: pushes low poly surface out (alternative to distance)
9. **Click Bake** → Blender projects high poly detail onto the image
10. **Save image** → UV/Image Editor → Image → Save As

### Baking Settings Details
- **Ray Distance**: Too low = missing details. Too high = capturing inside/behind.
- **Cage**: Duplicate low poly, scale up slightly = precise ray origin control
- **Max Ray Distance**: Limit how far rays travel (prevent far captures)
- Use Cycles engine (not Eevee) for best baking quality

### Using the Normal Map
1. Add **Normal Map** node to material
2. Set image color space to **Non-Color** (NOT sRGB!)
3. Connect Image Texture → Normal Map node → Principled BSDF Normal input
4. Adjust **Strength** (usually 0.5-1.0)
5. Normal map creates fake surface bumps without geometry

### Other Bake Types
- **Ambient Occlusion** (AO): Cavity darkening, natural ambient shadows
- **Diffuse/Base Color**: Bake painted colors to texture
- **Roughness/Metallic**: Bake material properties
- **Displacement**: True height data for render-time subdivision
- **Combined**: Everything in one go (for static renders)

## 5. RIGGING — ARMATURES, WEIGHT PAINTING & IK/FK

### What is Rigging
- Adding a digital skeleton (armature) to a mesh so it can deform/animate
- Bones control vertices via vertex groups/weight painting

### Creating an Armature
1. Add → Armature → Single Bone
2. Tab into Edit Mode → extrude bones (E key) to build skeleton
3. **Naming convention matters**: `UpperArm.L`, `UpperArm.R` for symmetry
4. **Bone rolls** (Ctrl+R) control twist orientation

### Parenting Mesh to Armature
- **With Empty Groups**: Creates vertex groups matching bone names, you paint weights manually
- **With Automatic Weights**: Blender guesses weights (quick start, needs cleanup)
- **With Envelope**: Uses bone influence radius (rarely used)
- **Armature Modifier** added to mesh → set target = armature

### Weight Painting
- **Weight Paint Mode**: Paint how much each bone influences each vertex
- Red = 1.0 (full influence), Blue = 0.0 (no influence), Green/Yellow = partial
- **Tools**: Draw, Blur, Sharpen, Darken/Lighten
- **Normalize**: Ensures total influence per vertex = 1.0
- **Mirror**: Paint on one side, mirrors to other (X-Axis)

### Weight Painting Best Practices
- Smooth gradients between bones (no hard cuts)
- Check deformations by rotating bones in Pose Mode
- Common problem: "Candy wrapper" twist → fix with correct bone rolls
- Use **Vertex Group** panel to lock groups while painting others

### IK (Inverse Kinematics) vs FK (Forward Kinematics)
- **FK**: Rotate bones directly (shoulder → upper arm → forearm → hand). Like a stop-motion puppet.
- **IK**: Move a "target" controller and bones bend automatically (hand position controls arm). Like grabbing a puppet's hand and the arm follows.
- **IK Constraint**: Add to last bone → set target = empty/mesh controller
- **Pole Target**: Controls elbow/knee direction (prevents backwards bending)
- **Switching**: Advanced rigs have IK/FK switching (animate in IK, fine-tune in FK)

### Rigify (Built-in Addon)
- Generate complete human rig from metarig
- Includes: spine, arms, legs, fingers, face, IK/FK switching
- Metarig → adjust to match character proportions → Generate Rig
- Much faster than manual rigging for humanoids

### Rigging for Games
- **Bone count matters**: Keep under 256 for mobile, 1024 for desktop
- **Weight limit**: Max 4 bones per vertex (standard for game engines)
- **Root bone**: One bone at origin for moving entire character
- **Apply transforms** (Ctrl+A → All Transforms) before rigging

## 6. GEOMETRY NODES — PROCEDURAL WORKFLOW

### What Are Geometry Nodes
- Visual scripting for geometry manipulation
- Node-based workflow (like shader nodes but for 3D data)
- Non-destructive: change inputs, whole system updates
- Fields system: data flows through nodes as "fields" (attributes)

### Essential Geometry Nodes
- **Distribute Points on Faces**: Scatters points on mesh surface
- **Instance on Points**: Places objects at each point
- **Set Position**: Moves geometry based on math/fields
- **Join Geometry**: Combines multiple geometry streams
- **Separate Geometry**: Splits by condition
- **Random Value**: Per-point randomization
- **Attribute Statistic**: Get min/max/mean of attributes
- **Ray Cast**: Shoot rays from geometry, detect hits
- **Geometry Proximity**: Find distance to target geometry

### Common Geometry Nodes Workflows

**Procedural Scattering**:
1. Distribute Points on Faces → density based on surface area
2. Instance on Points → place tree/rock/flower objects
3. Random Value → rotate/scale each instance uniquely
4. Weight Paint vertex group → control density by painting

**Procedural Modeling**:
1. Start with basic shape (cube, circle)
2. Extrude → bevel → set position nodes
3. Use math nodes to drive parameters
4. Animate group inputs for motion graphics

**Data Flow in Fields**:
- Each point/face/edge carries "attributes" (position, normal, custom)
- Nodes read and write attributes
- "Realizing" instances converts them to real geometry

### Geometry Nodes for Environment Art
- Scatter vegetation on terrain
- Create procedural city blocks
- Generate roads following curves
- Place rocks with random rotation and scale
- Wind animation via noise-driven displacement

### Geometry Nodes Modifier
- Applied like any modifier (Modifier Properties → Add Modifier → Geometry Nodes)
- Group inputs = exposed sliders you can keyframe
- Encapsulate complex setups → reuse on any object

## 7. ANIMATION — KEYFRAMES, GRAPH EDITOR & CONSTRAINTS

### Keyframe Basics
- **I key**: Insert keyframe (location, rotation, scale, or all)
- **Timeline**: Shows frames, playhead, keyframe markers
- **Dope Sheet**: Overview of all keyframes across objects
- **Graph Editor**: Fine-tune animation curves (ease in/out, overshoot)
- **Auto Keyframe**: Red circle in timeline → records changes automatically

### The 12 Principles of Animation (Blender Application)
1. **Squash and Stretch**: Scale objects during motion (bounce = squash on impact, stretch during fall)
2. **Anticipation**: Wind-up before action (crouch before jump)
3. **Staging**: Clear silhouettes, camera placement
4. **Straight Ahead/Pose to Pose**: Plan key poses, fill in-between
5. **Follow Through**: Hair/clothes continue after body stops
6. **Slow In/Slow Out**: Easing curves in Graph Editor (not linear)
7. **Arcs**: Natural motion follows curves, not straight lines
8. **Secondary Action**: Blinking, breathing, clothing details
9. **Timing**: More frames = slower, fewer = faster
10. **Exaggeration**: Push poses beyond real life for impact
11. **Solid Drawing** (3D equivalent): Good form, weight, balance
12. **Appeal**: Character design that draws attention

### Graph Editor Workflow
1. Insert keyframes in timeline
2. Open Graph Editor (Shift+F6)
3. Select curves (location X/Y/Z, rotation)
4. **T key**: Set keyframe interpolation:
   - Constant: Holds until next keyframe (snappy)
   - Linear: Constant speed (mechanical)
   - Bezier: Smooth ease in/out (natural)
5. **V key**: Handle type (auto, vector, aligned, free)
6. **Shift+Ctrl+click**: Add keyframe to curve
7. Smooth curves for organic motion, sharp for mechanical

### Constraints for Animation
- **Copy Location/Rotation**: One object follows another
- **Damped Track**: Object points toward target (like eyes at player)
- **Track To**: Hard lock on target
- **Limit Location/Rotation**: Prevent passing thresholds
- **Child Of**: Complex parenting with influence slider
- **Follow Path**: Move along curve (for vehicles, cameras)
- **Shrinkwrap**: Keep object on surface

### Character Animation Pipeline
1. **Planning**: Thumbnails → key poses (storyboard)
2. **Blocking**: Keyframes at key poses (every 8-12 frames)
3. **Splining**: Fill in-betweens, smooth curves
4. **Polishing**: Add overlap, follow-through, micro-movements
5. **Rendering**: Playblast for review

### Shape Keys (Morph Targets)
- **What**: Store alternate mesh shapes (facial expressions, muscle flex)
- **Basis**: Default shape
- **Key 1, Key 2...**: Alternate shapes
- **Value 0-1**: Morph between basis and key shape
- **Drivers**: Link shape key value to bone position (smile when mouth bone moves)
- **Game use**: Export as morph targets for real-time facial animation

### Animation for Games
- **Actions**: Named animation clips (Idle, Walk, Run, Attack)
- **NLA Editor**: Non-Linear Animation → layer/blend actions
- **Looping**: Match first and last frames (walk cycle)
- **Root motion**: Move character via animation (not code), extracted from hip bone
- **Export**: FBX with baked animation, or glTF (better for web)

## 8. SHADERS & MATERIALS DEEP DIVE

### Principled BSDF Parameters
- **Base Color**: Albedo (surface color without lighting)
- **Metallic**: 0 = dielectric (plastic, rock), 1 = metal (gold, steel)
- **Specular**: Replaced by IOR in 4.0+ (index of refraction)
- **Roughness**: 0 = mirror, 1 = matte (microsurface scatter)
- **IOR**: Index of refraction for glass/water (1.45 glass, 1.33 water)
- **Alpha**: Transparency (0 = invisible, 1 = opaque)
- **Normal**: Bump/normal map input for surface detail
- **Subsurface**: Light scattering inside surface (skin, wax, marble)
- **Emission**: Self-illuminating (glow in dark)
- **Clearcoat**: Car paint top layer (1 = full clearcoat)
- **Clearcoat Roughness**: How glossy the clearcoat is
- **Sheen**: Velvet-like rim light effect

### Special Material Types
- **Glass**: Principled BSDF, transmission 1.0, IOR 1.45
- **Water**: Similar to glass, IOR 1.33
- **Skin**: Subsurface scattering, subsurface radius (R, G, B different penetration)
- **Emissive**: Emission shader (not Principled) for pure light sources
- **Hair**: Principled Hair BSDF (anisotropic highlight, melanin controls)
- **Volume**: For fog, clouds, god rays (absorption/scattering)

### Node Groups (Reusable Shaders)
- Select nodes → Ctrl+G → Group
- Expose parameters as group inputs
- Use across multiple materials
- Example: "Car Paint" group with exposed color/flake/clearcoat parameters

## 💡 KEY INSIGHTS & TAKEAWAYS

### The Big 3 Rules
1. **Never apply modifiers early** — keep them live until final export
2. **Topology matters for animation** — quads, edge flow, pole placement
3. **Reference first, model second** — always have images open

### The Most Important Skill
- **Not any single technique** — it's knowing which technique to use WHEN:
  - Boolean workflow for mechanical/hard surface
  - Sculpting for organic/character
  - Geometry nodes for environment repetition
  - Procedural textures for variation without UV work

### Performance Hierarchy (Fastest → Slowest)
1. Simple solid color material (no textures)
2. Procedural textures (GPU calculates)
3. Image textures with UVs (memory bandwidth)
4. Subsurface scattering (ray bounces)
5. Volumes (most expensive)

### File Size Optimization
- Use ORM maps (Ambient Occlusion + Roughness + Metallic in RGB channels)
- Compress textures (KTX2, basis universal for web)
- Bake everything possible (AO, lighting, normals)
- Remove unused UV channels
- Use PNG for masks, JPG for color, EXR for HDR

### The $MAD Application
- **Car paint material** = procedural with noise flake, clearcoat, emissive neon strips
- **MAD Chao character** = sculpt base in dyntopo → retopo → multires detail → bake normal → rig with simple armature → animate idle breathing
- **Track environment** = geometry nodes scatter for debris, rocks, banners
- **Rim lighting** = emission strips on car edges for that neon glow
- **Normal map detail** = baked panel lines, vents, bolts from high poly to low poly car body
