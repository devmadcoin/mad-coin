## What I Learned Today

### The Big Insights
1. **Design is psychology, not decoration.** Every pixel is a decision about what the user will feel and do. The aesthetic-usability effect means beautiful interfaces are PERCEIVED as more usable — $MAD's dark aesthetic is functional, not just decorative.

2. **Dark mode is a science, not a trend.** 82% of users prefer it. Pure black (#000000) is wrong — use layered dark grays (#121212, #1A1A1A, #242424). The $MAD website's white/45 and white/55 opacity system is textbook dark mode hierarchy. But I need to verify it passes WCAG AA contrast ratios.

3. **The 5-second rule is brutal.** A user should understand what your site does in 5 seconds without scrolling. The $MAD homepage needs to answer: What is $MAD? Why does it matter? What do I do next? — all above the fold.

4. **Crypto UX is famously terrible.** The $MAD opportunity: be the exception. Jargon kills adoption. "Gas fee" → "Network fee." "Slippage" → "Price may change slightly." Every wallet connection, transaction flow, and explanation should feel human.

5. **Accessibility is not optional — it's legally required and ethically right.** 80.3% of websites fail contrast checks. ADA, EAA, UK Equality Act all reference WCAG 2.1 AA. 15% of the world has a disability. Design for everyone.

### Key Principles Added to Knowledge
- Jakob's Law: Users want your site to work like other sites. Innovation must be 10x better to justify breaking convention.
- Hick's Law: More choices = paralysis. 3 pricing tiers, not 12. 3 wallet options, not 20.
- Fitts's Law: Big, close targets are easier to hit. Primary CTAs must be large and in thumb zones.
- The Paradox of Choice: 24 jams = 3% purchase. 6 jams = 30% purchase. Reduce, reduce, reduce.
- WCAG Contrast: 4.5:1 for body text, 3:1 for large text/UI elements. 80.3% of sites fail this.
- Mobile-first: 55%+ of traffic is mobile. Design for the smallest screen first.
- Micro-interactions: 200-500ms, 60fps, purposeful. Every animation must earn its place.
- Design tokens: `color-background` = swap values for light/dark mode. Change once, update everywhere.

### $MAD-Specific Applications
- **Website**: The hero section needs to pass the 5-second test. One primary CTA per page. Contract address must be copyable everywhere. Loading states need skeleton screens, not blank spaces.
- **Telegram Bot**: Inline keyboards reduce cognitive load (Hick's Law). The bot's personality IS the brand — every message is a touchpoint. Response time <1 second.
- **X Posts**: Dark mode native design. Visual posts perform 2-5x better than text. Every post should be understood in 1-2 seconds of scrolling.
- **Dark Mode Audit**: The current white/45% opacity text needs to be checked against WCAG AA on the actual background. If it fails, bump to 50-55%.
