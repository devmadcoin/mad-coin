
## 2026-07-01 - Holder Verification Tool Built for Rewards Page

### Context
User (MAD Dev) asked me to analyze and replicate a community-built $MAD holder verification tool:
- URL: `https://mad-holder-check.netlify.app/`
- Built by a MAD community member
- Read-only wallet verification (no connection needed)
- Checks if wallet qualifies for giveaway based on first hold date and balance

### Tech Analysis of Community Tool
- Single-file HTML/CSS/JS — no frameworks, deployed to Netlify
- Solana RPC `getTokenAccountsByOwner` + `getSignaturesForAddress` to trace first hold date
- Google Sheets logging via `fetch` with `no-cors`
- Dev config panel with hardcoded password (`madrich`)
- Categorizes holders: Old Holder (before cutoff), New Holder (within window), Too Late (after window), Not Qualified (below minimum)
- Default: 1,000 MAD minimum, 50 old + 30 new spots at 10M market cap

### What I Built
Integrated a full **HolderVerification** component into `/rewards` page at `https://mad-coin.vercel.app/rewards`:

**Features:**
- Paste wallet address + Telegram username → checks on-chain
- Queries Solana RPC for MAD token balance and first hold date
- Category assignment with badges: Old Holder (gold), New Holder (green), Too Late (red), Not Qualified (gray)
- Stats breakdown: Balance, Hold Date, Category, Minimum Required
- Contextual message explaining why they do/don't qualify
- "How It Works" 3-step explainer (Paste → Check → Get Rewarded)
- Full dark theme matching the rewards page (red `#FF2D2D` accents, black cards, white text)
- Read-only: no wallet connection, no signing, no approvals

**Solana Integration:**
- Uses `https://api.mainnet-beta.solana.com` RPC endpoint
- `getTokenAccountsByOwner` with MAD mint: `Fa7ZE9nCEYnrHsnoeHuhEExJpchtrBtKXnWe6CgHpump`
- `getSignaturesForAddress` to trace oldest transaction and determine first hold date
- Proper error handling for invalid addresses, non-holders, network failures
