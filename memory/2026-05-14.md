## 2026-05-14 Session — X Bot Repetition Fix

**Problem:** User caught the X bot posting the exact same thread repeatedly. Screenshot showed "The $MAD framework in 3 parts" posted 4 times with declining engagement (Part 1: 57 views → Part 2: 13 views).

**Root cause:** `mad_x_bot.py` only had **2 thread templates** in `generate_thread_candidates()`. With `AUTO_POST_THREADS=true`, the bot cycled between just 2 threads endlessly. No deduplication existed for threads.

**Fix applied:**
1. Expanded thread pool from **2 → 8 unique threads** (jeeter thesis, conviction signs, $MAD as filter, etc.)
2. Added **thread deduplication**: checks first 50 chars against last 30 posted texts, skips if recently used
3. Added **hard similarity block** in `generate_philosophy_candidates()`: `combined_similarity() > 0.60` → blocked entirely (not just penalized)
4. Added **20+ overused template phrases** to `OVERUSED_TEMPLATE_PATTERNS` (exact lines that were repeating)
5. Pushed to main branch — commit `aef3e2c`

**Lesson:** Small template pools + no deduplication = algorithm death spiral. Engagement drops compound when X detects repetition.
